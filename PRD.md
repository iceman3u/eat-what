# 《吃什么》AI-native 项目需求说明文档（最终版）

## 一、项目概述

### 1.1 项目名称
- 中文名：吃什么
- 英文名：EatWhat

### 1.2 项目背景
很多人在每天吃饭时会出现选择困难、决策疲劳，或者想自己做饭但缺乏灵感。本项目旨在打造一个轻量、快速、高颜值的 AI-native 美食决策应用，帮助用户快速决定今天吃什么。

### 1.3 核心产品定位
《吃什么》是一款以“帮助用户快速解决吃饭决策”为核心目标的高颜值轻量 Web 应用。产品强调：
- 快速
- 随机感
- 情绪价值
- 移动端体验
- 极简交互

第一阶段以移动端 H5 为核心，兼容桌面端，后续扩展微信小程序。重点是“帮助用户快速决定吃什么”，而不是复杂的专业菜谱平台。

---

## 二、功能需求

### 2.1 核心功能
1. 随机推荐菜品
2. 展示菜品图片（静态图库，不允许用户 AI 生成图片）
3. 显示菜品基本信息：名称、难度、用时、热量、人均价格
4. 展示简单做法步骤
5. 支持“换一个菜品”按钮
6. 移动端优先，桌面端兼容

### 2.2 非核心功能（未来升级）
- 天气/时间/用户偏好推荐
- 冰箱模式：根据用户食材推荐菜品
- AI 个性化推荐（后期系统统一生成菜品图片）

### 2.3 禁止功能
- 用户直接 AI 生菜品图片
- 复杂表单、评论、收藏等社交功能
- 原生 App（第一阶段）

---

## 三、用户流程

1. 用户打开 H5 页面
2. 点击“今天吃什么”按钮
3. 系统随机推荐菜品
4. 展示菜品图片和基本信息
5. 展示做法步骤
6. 用户可点击“换一个菜品”重新获取

## 四、界面规格与设计

### 4.1 首页
- 大按钮：“今天吃什么”
- 极简、治愈风格，动画滚动体验

### 4.2 菜品展示页
- 菜品大图
- 菜品名称
- 信息卡：时间、难度、热量、人均价格
- 做法步骤列表
- “换一个菜品”按钮
- UI 动效突出随机感与轻松感

### 4.3 风格与体验
- 高颜值、极简、轻量
- 治愈感、轻松感
- 卡片式交互、动画过渡

## 五、技术方案

### 5.1 技术栈
- 前端：Next.js 15 + React + TypeScript + TailwindCSS + Framer Motion
- 后端：Next.js API Routes（Route Handlers）
- 数据库：PostgreSQL + Prisma ORM
- 静态资源：dish-images/ 存放菜品图片
- 部署：GitHub + Vercel（自动部署）

### 5.2 项目结构
```
eat-what/
│
├── app/                    # Next.js App Router
├── components/             # UI 组件
├── lib/                    # 工具库
├── services/               # API 服务
├── prisma/                 # Prisma schema
├── public/
│   └── dishes/             # 静态菜品图片
├── types/
├── hooks/
├── styles/
├── docs/
├── CLAUDE.md               # Claude Code 项目规则
├── .claude/
│   ├── skills/
│   ├── hooks/
│   └── mcp/
├── package.json
└── README.md
```

### 5.3 部署方案
- GitHub 代码仓库
- Vercel 自动部署
- 域名绑定（例如 eatwhat.app）
- Railway PostgreSQL 数据库

### 5.4 Claude Code 指引
- 使用 CLAUDE.md 记录项目规则：
  - TypeScript 全栈统一
  - 静态菜品图库
  - 禁止用户 AI 生图
  - 移动端优先
  - 高颜值轻量风格
- Skills 可定义核心功能模块，如随机推荐、做法步骤生成
- Subagent 可拆分任务，如前端 UI、API、数据处理
- MCP 用于未来接入外部接口（如天气或食材库）
- Hooks 用于自动格式化、测试、部署触发
